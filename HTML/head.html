What's in the head? Metadata in HTML

El head de HTML es la parte del documento que no se muestra al usuario. En esta parte se carga la informacion como el titulo, el autor, una breve descripcion, links al CSS, etc. Los navegadores usan el head del HTML para interpretar correctamente el contenido del archivo.

What is the HTML head?

Veamos este ejemplo: 

      
        <!doctype HTML>
        <html lang="en-US">
          <head>
            <meta charset="utf-8" />

            <title> Mi test page </title>
          </head>

          <body>
            <p> This is my page </p>
          </body>
        </html>

A diferencia del body, lo que se agrega en el head del documento no se muestra al usuario. El trabajo del head es contener metadatos del documento, en el ejemplo anterior el head es muy pequeno, pero puede ser mas extenso en paginas mas grandes.

Addin a title

Ya vimos el elemento de <title> en accion y sabemos que este es usado para agregar un titulo a la pagina. A menudo este es confundido con el elemento <h>, pero ambos tienen una funcion totalmente diferente.

  • El elemento <h1> se usa para marcar el encabezado principal de la pagina, se suele usar solo uno por pagina, entonces lo mas comun es que este marque el titulo principal del contenido.

  • Por otro lado el elemento <title> forma parte de los metadatos y representa el titulo de todo el documento y no del contenido del documento. El elemento de title tambien tiene otras funciones, por ejemplo cuando queremos guardar una pagina y hacemos un bookmark en la seccion del nombre recomendado nos saldra el texto que se encuentre encerrado entre el elemento <title>.

Metadata: the <meta> element

Los metadatos son datos de datos. HTML tiene su forma oficial para anadir metadatos a el documento, el cual es el elemento de <meta>. Todo de lo que hablaremos en esta seccion son metadatos. En realidad hay una grand variedad de tipos de metadatos que podemos agregar al head de nuestra pagina.

Specifying your document's character encoding

En los ejemplos anteriores hemos visto esta linea incluida:

<meta charset="utf-8" /gt;

Este elemento especifica la codificacion de caracteres del documento, o sea el set de caracteres que puede usar el documento. El set utf-8 incluye la gran mayoria de todos los caracteres del lenguaje humano por lo que tu pagina podra contener la mayoria de lenguajes humanos.

NOTA!!: Algunos navegadores corrigen automaticamente una codificacion incorrecta pero de igual forma es mejor incluirla para evitar problemas.

Adding an author and description

Algunos elementos <meta> contienen los atributos de name y description:

  • El atributo de name para especificar que tipo de informacion se contiene ahi.

  • El elemento content especifica el metacontenido real.

A continuacion tenemos un ejemplo de como pueden ser usados estos dos atributos: 

      
        <meta name="author" content="Chris Mills" />
        <meta
          name="description"
          content="The MDN Web Docs Learning Area aims to provide
          complete beginners to the Web with all they need to know to get
          started with developing websites and applications." />

Especficar el autor del contenido tiene muchos beneficios. Un ejemplo de esto es el de un usuario que tiene dudas sobre la pagina, algunos manejadores de contenido tienen funciones para extraer el autor de una pagina y asi saber a quien contactar

Por otro lado tenemos la descripcion, poniendo una descripcion a nuestra pagina con una buena redaccion puede generar que tu pagina aparezca mas arriba en los buscadores, a este tipo de actividades normalmente se les conoce como Search Engine Optimization (SEO).

Other types of metadata

A medida que navegues por la web te iras encontrando con diferentes tipos de metadatos, algunos de estos son creaciones de otras empresas para proveer metadatos mas nutridos.

Un ejemplo de esto es el Open Grap Data el cual es un protocolo de metadatos creado Por Facebook dar a los sitios web metadatos mas nutridos, por ejemplo en la pagina de MDN Web Docs encontraras esto: 

      
        <meta
          property="og:image"
          content="https://developer.mozilla.org/mdn-social-share.png" />
        <meta
          property="og:description"
          content="The Mozilla Developer Network (MDN) provides
          information about Open Web technologies including HTML, CSS, and APIs for both websites
          and HTML Apps." />
        <meta property="og:title" content="Mozilla Developer Network" />

Un efecto de esto es que cuando ponemos un link en Facebook hacia MDN Web Docs el link aparecera junto con una imagen y una description para una mejor experiencia del usuario.
