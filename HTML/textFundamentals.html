HTML text fundamentals

Uno de los principales trabajos de HTML es darle la estructura a la pagina segun sea la intencion de los desarrolladores.

The basic: headings and paragraphs

La mayoria de los textos estructurados constan de encabezados y parrafos para asi hacer mas sencilla la experiencia del usuario. En HTML cada parrafo va encerrado entre el elemento <p>:

<p> I am a paragraph, oh yes I am </p>

Cada encabezado va encerrado entre el elemento de encabezado:

<p> I am the title of the history </p>

Hay varios tipos de headers: h1, h2, h3, h4, h5 y h6. Cada uno de estos se usa para representar diferentes niveles en el documento. h1 representa el titulo principal, h2 los subtitulos, h3 los sub subtitulos y asi sucesivamente.

Implementing structural hierarchy

Por ejemplo en la siguiente historia <h1> representa el titulo de la historia, <h2> representa el titulo de cada capitulo y <h3> las secciones de cada capitulo. 

            
                <h1>The Crushing Bore</h1>

                <p>By Chris Mills</p>

                <h2>Chapter 1: The dark night</h2>

                <p>
                    It was a dark night. Somewhere, an owl hooted. The rain lashed down on the…
                </p>

                <h2>Chapter 2: The eternal silence</h2>

                <p>Our protagonist could not so much as a whisper out of the shadowy figure…</p>

                <h3>The specter speaks</h3>

                <p>
                    Several more hours had passed, when all of a sudden the specter sat bolt
                    upright and exclaimed, "Please have mercy on my soul!"
                </p>

Realmente depende de uno mismo que es lo que querramos que signfiquen los elementos, siempre que tengan sentido y jerarquia. Pero a continuacion se presentan algunas de las mejores practicas:

  • Preferiblemente solo debe tener un <h1> por pagina, este es el "top level heading" y todos los demas van debajo de este para respetar la jerarquia.

  • Asegurate de seguir correctamente la jerarquia, es decir, que no puedes usar <h3> para los subtitulos y <h2> para los sub subtitulos

  • De los seis niveles de header que existen no deberias usar mas de tres por pagina ya que con muchos niveles de texto se vuelve dificil la lectura. En algunas ocasiones es recomendable repartir este contenido en varias paginas.

Why do we need structure?

Veamos un ejemplo de un documento de html que contiene las instrucciones para hacer un hummus.

Podemos observar que todo el texto aparece como un gran trozo de informacion, esto es porque no hay elementos que le indiquen al browser como estructurar el contenido. Entonces:

  • Los usuarios tienden a escanear rapido la pagina para encontrar informacion rapido por lo que usualmente solo leen los encabezados. Entonces si no encuentran nada util abandonan la pagina.

  • Para aplicar CSS o JavaScript a tu pagina necesitas tener los elementos encerrados correctamente para que asi se puedan encontrar de una manera mas facil.

Why do we need semantics?

La semantica es una rama de la linguistica que se centra en entender como las palabras y las estructuras linguisticas transmiten un significado. Por ejemplo, cuando vemos que en un semaforo hay luces rojas sabes que esto es un "STOP" y que unas luces verdes indican "GO". Las cosas pueden parecer muy confusas si se aplica una semantica incorrecta

De manera similar tenemos que saber usar los elementos de una forma correcta para darle al contenido una estructura y funcionamiento optimo. En el siguiente ejemplo encierra el texto en el elemento <h1>, el cual es un elemento semantico que le da un rol de top level heading al texto que esta encerrado en el.

<h1> This is a top level heading <h1>

Por defecto el browser le asignara un tamano de fuente grande, claro que podemos cambiar esto en cualquier momento con CSS. Lo mas importante es que al texto encerrado se le ha dado un valor semantico que puede ser usado por los screen readers o los motores de busqueda.

Otra manera de hacer parecer un texto un top level heading es asi:

<span style="font-size: 32px; margin: 21px 0; display: block;"> Is this a top level heading? </span>

Este elemento es llamado span. No tiene semantica. Es usado para encerrar elementos a los que se les quiere aplicar CSS o JavaScript sin darle un significado semantico extra. Puedes usar CSS y hacer que luzca como un top level heading pero no contara con las propiedades semanticas que tiene el elemento <h1>.

Lists

Las listas se encuentran en todos lados, desde tu lista de compras de supermercado o la lista de instrucciones que sigues para llegar a tu casa. Hay tres tipos de listas sobre las que nos tenemos que preocupar.

Unordered

Este tipo de listas se usan cuando queremos marcar elementos en los cuales no importa su orden, como por ejemplo la lista de compras. Para crear este tipo de listas se usa el elemento <ul> (unordered list) y cada elemento que se anide dentro de esta lista se colocan dentro de el elemento <li> (list item): 

            
                <ul>
                    <li> milk </li>
                    
                    <li> eggs </li>

                    <li> bread </li>

                    <li> hummus </li>
                </ul>

Ordered

En este tipo de listas el orden si importa, por ejemplo se usan para las direcciones para llegar a un lugar, este tipo de listas se encierran entre el elemento <ol>, e igual con las unordered list cada elemento tiene que estar encerrado en el elemento <li>: 

            
                <ol>
                    <li> Drive to the end of the road <li>

                    <li> Turn right </li>

                    <li> Go straight across the first two roundabouts </li>

                    <li> Turn left at the third roundabout </li>

                    <li> The school is on your right, 300 meter up the road </li>
                </ol>

Description

Este tipo de listas encierran grupos de terminos y descripciones, es decir pares clave-valor, Estas listas se encierran entre el element <dl>, el termino se encierra entre <dt> y las descripciones entre <dd>. 

            
                <dl>
                    <dt> Albacore Tuna </dt>
                    <dd> The albacore is a ver fast swimmer </dd>

                    <dt> Asian carp </dt>
                    <dd> Asian carp can consume 40% of their body weight in food a day </dd>

                    <dt> Barracuda </dt>
                    <dd> Can grow to nearly 2 meters long! </dd>
                </dl>

Nesting lists

Es algo aceptable indexar una lista dentro de otra, por ejemplo en esta receta hay un paso que tiene dos sub-pasos: 

            
                <ol>
                    <li> Remove the skin of the garlic, and chop coarsely </li>

                    <li> Remove all the seeds and stalk from the pepper, and chop coarsely </li>

                    <li> Add all the ingredients into a food processor </li>

                    <li> Process all the ingredients into a paste </li>

                    <ul>
                        <li> If you want a coarse "chunky" hummus, process it for a short time </li>

                        <li> If you want a smooth hummus, process it for a longer time </li>
                    </ul>
                </ol>

Emphasis and importance

En el lenguaje humano solemos enfatizar algunas palabras para cambiar el significado de algun enunciado, o algunas veces marcamos ciertas palabras que consideramos importantes. HTML provee varios elementos semanticos para lograr esto.

Emphasis

Cuando buscamos agregar enfasis en el lenguaje hablado, enfatizamos algunas palabras para alterar el significado de lo que decimos. De la misma forma en el lenguaje escrito enfatizamos algunas palabras colocandolas en italicas. Por ejemplo estos dos enunciados tienen diferentes significados:

  • I am glad you were'nt late.
  • I am glad you were'nt late.

En el primer enunciado se puede entender que la persona en realidad no llego tarde, pero en la segunda cuando se hace enfasis con la palabras suena sarcastica expresando que la persona si llego un poco tarde.

En HTML usamos el elemento <em> para lograr esto. Para hacerlo mas interesante este elemento tiene un valor semantico que los screen readers pueden cambiar el tono de voz en estas palabras. Los browsers le aplican un estilo de italicas a lo que se encuentre encerrado en el elemento <em> pero no necesariamente es una italica pura, para dar este estilo se usaria el elemento <i> o el elemento <span> y algo de CSS.

Strong importance

Para enfatizar palabras importantes solemos enfatizar en el lenguaje hablado y ponerlos en negritas en el lenguaje escrito. Por ejemplo:

  • This liquid is highly toxic.
  • I am counting on you. Do not be late!

En HTML usamos el elemento <strong> para lograr esto, este elemento tambien cuenta con propiedades semanticas por lo que los screen readers tambien cambiaran su tono de voz al leer estas secciones de texto. Los browsers automaticamente pondran estas secciones en negritas, pero no debemos usar <strong> si queremos un estilo de negritas puro, en ese caso usaremos el elemento <span> y algo de CSS o el elemento <b>.

Italic, bold, underline...

La situacion con los elementos <i>, <b> y <u> es mas complicada, estos eran muy usados en una epoca en la que CSS era poco o nada soportado, elementos como estos son llamados presentational elements ya que solo cambian la presentacion y no la semantica. Estos ya no son usados ya que en la actualidad la accesibilidad y el SEO es mas importante.

En HTML5 se redefinieron los roles de estos elementos. Solo es apropiado usar estos tres elementos anteriores cuando no hay una mejor opcion (usualmente si la hay). Considera usar <strong>, <em>, <mark> o <span>.

Siempre ten en mente la accesibilidad, por ejemplo el elemento <i> no es muy util para alguien que esta usando un screen reader

  • <i> es usado con palabras foraneas, terminos tecnicos, designacion taxonomica, etc
  • <b> es usado con palabras clave, nombres de productos o enunciados principales.
  • <u> se usa con nombres propios, errores de ortografia, etc.

La gente comunmente asocia el subrayado con los links. En la web es mejor usarlo solo para los links pero si es semanticamente apropiado usarlo en otras partes considera usar CSS para cambiar el estilo predeterminado.
