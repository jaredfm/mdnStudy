Getting started with HTML

What is HTML?

HTML (HyperText Markup Language) es un lenguaje de marcado que se utiliza para darle estructura a las paginas web. Este consiste en una serie de elementos los cuales son usados para que las diferentes partes de contenido de una pagina web se comporten o aparezcan de una manera especifica, por ejemplo se pueden hacer hyperlinks a otras paginas o poner fragmentos de texto en italicas. Por ejemplo, considera el siguiente texto.

My cat is very grumpy

Si quisieramos que el texto se mantuviera por si solo, tendriamos que encerrarlo dentro del elemento de parrafo <p>:

<p> My cat is very grumpy <p>

NOTE!!: Los tags en HTML no distinguen entre minusculas o mayusculas (case-sensitive), por lo que es lo mismo escribir <title>, <Title>, <TITLE> o <TiTle> y esot seguira funcionando, sin embargo es mejor mantener los nombres de los tags en minusculas para mantener la legibilidad y la consistencia.

Anatomy of an HTML element

Vamos a explorar mas a fondo el elemento del parrafo que se presento anteriormente

La anatomia del elemento es la siguiente:

  • Opening tag: Este consiste en el nombre del elemento, encerrado entre los simbolos de menor y mayor que, despues de este opening tag es donde elemento comienza a tener efecto. En el caso anterior el nombre es p, de paragraph

  • Contenido: Es lo que se encuentra dentro del elemento, en este caso es el texto "My cat is very grumpy"

  • Closing tag: Esto es lo mismo que el opening tag, solo que antes del nombre del elemento se coloca un slash "/", este indica que aqui dejara de tener efecto el elemento. En el caso anterior el closing tag es </p>.

Entonces la anatomia completa de un elemento es opening tag, contenido y closing tag, en ese orden

Nesting elements

Los elementos pueden contener otros elementos dentro de ellos, a esto se le llama anidacion (nesting). Por ejemplo, si en la frase anterior quisieramos resaltar la palabra "very" tendriamos que encerrarla en el elemento de <strong> de la siguiente forma:

< My cat is <strong> very <strong> grumpy >

Lo que nos daria como resultado lo siguiente:

My cat is very grumpy

Esa es la forma correcta de hacer nesting, los elementos que se abrieron dentro de otro elemento se tienen que cerrar primero

Void elements

No todos los elementos siguen el patron de opening tag, content y closing tag, hay elementos que consisten en un solo tag, los cuales se usan tipicamente para insertar contenido dentro del documento, a estos elementos se les llama void elements, un ejemplo de estos elementos es el elemento <img>, el cual se usa para insertar imagenes:

<img src="https://raw.githubusercontent.com/mdn/beginner-html-site/gh-pages/images/firefox-icon.png" alt ="Firefox icon" />

Attributes

Los atributos contienen informacion extra del elemento que no se muestra en el contenido, por ejemplo el atributo class se usa como id para luego ubicar al elemento con informacion de estilos

Un atributo debe tener:

  • Un espacio que lo separe del nombre del elemento y en el caso que que exista mas de un atributo deben estar separados por espacios

  • El valor del atributo encerrado entre comillas dobles

Boolean attributes

En ocasiones te toparas con atributos los cuales no tienen ninguna valor, esto es mas que aceptable, estos son llamado Boolean attributes. Estos pueden tener solo un valor, el cual por lo general suele ser el mismo que el nombre del atributo. Por ejemplo, consideremos el atributo "disabled", el cual se usa para que el usuario no pueda ingresar algo en alguna entrada de un formulario.

Omitting quotes around attribute values

Si te pones a mirar el HTML de otras paginas, te encontraras con diferentes estilos de marcado, incluyendo atributos en los cuales el valor no esta encerrado entro comillas. Esto es permitido en ciertas ocasiones, pero puede generar problemas en otras. Para entender mejor veamos un ejemplo: el elemento Anchor <a> se usa para convertir un fragmento de texto en un link y el atributo href se usa para indicar hacia donde apunta el texto.

<a href=https://www.mozilla.org> Favorite website </a>

Pero el elemento anchor puede contener tambien el atributo title, el cual incluye una pequena referencia sobre la pagina anclada, si agregamos el atributo title al ejemplo anterior obtendriamos lo siguiente:

<a href=https://www.mozilla.org title=The Mozilla homepage> Favorite website </a>

Lo anterior nos crearia un problema, nos crea tres atributos, el primero es title el cual se le asigna el valor de "The", y se crean dos boolean attributes, Mozilla y homepage, lo anterior nos daria resultados no deseados, por lo que siempre hay que usar comillas

Single or double quotes?

Como ya mencionamos antes, los valores de los atributos se encierran entre comillas dobles, pero puede que en algunos casos se tope con HTML en el cual usen comillas simples, esto es totalmente valido por lo que cualquiera de los dos ejemplos siguientes es correcto

  • <a href="https://www.example.com"> A link to my example </a>

  • <a href='https://www.example.com'> A link to my example </a>

Tienes que tener mucho cuidado de no mezclar los dos tipos de comillas, con la misma con la que se empieza se debe terminar. Sin embargo si usas un tipo de comillas para abrir y cerrar el valor puedes usar el otro tipo de comilla dentro de las anteriores.

<a href="https://www.example.com" title="Isn't this fun?"> A link to my example </a>

Si queremos usar el mismo tipo de comillas dentro y fuera del valor tenemos que usar HTML entities, de lo contrario tendremos un error. El primer ejemplo es la forma incorrecta, y el segundo es la forma correcta de hacerlo usando HTML entities.

  1. <a href="https://www.example.com" title="An "interesting" reference"> A link to my example <a>

  2. <a href="https://www.example.com" title="An "interesting" reference"> A link to my example </a>

Como podemos observar, en el primer ejemplo el atributo title toma el valor de An y se rompen los alcances de las comillas

Anatomy of an HTML document

Los elementos de HTML por si solos no son muy utiles, ahora veremos como combinarlos para crear una pagina completa. 

    
      <!doctype HTML>
      <html lang="en-US">
        <head>
          <meta charset="utf-8" />

          <title> Mi test page </title>
        </head>

        <body>
          <p> This is my page </p>
        </body>
      </html>

Aqui tenemos lo siguiente:

  • <!DOCTYPE html>: Cuando HTML era joven el doctype servia como un link hacia una serie de reglas que debia tener un buen HTML. Antes asi solia verse un doctype:

    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

    Pero con la ultima version de HTML la minima cadena requerida por el doctype es !DOCTYPE html

  • <html></html>: Dentro de este elemento se aloja todo el documento de la pagina web, tambien es conocido como el elemento raiz (root element).

  • <head></head>: Este elemento actua como un contenedor para todo lo que quieras que contenga tu pagina. No contiene nada que se mostrara a los usuarios de la pagina. En el head se incluye la informacion de estilo, el titulo de la pagina, el set de caracteres o palabras clave que apareceran en los resultados de busqueda.

  • <meta charset="utf-8" />: El elemento <meta> se usa para representar metadatos que no pueden ser representados por otro elemento. El atributo charset se usa para especificar la codificacion de caracteres del documento, en este caso es UTF-8, el cual incluye la gran mayoria de los caracteres del lenguaje humano. No hay motivos para no incluir este elemento y puede ayudarnos a evitar futuros problemas.

  • <title></title>: Este elemento sirve para establecer el titulo de la pagina en la que nos encontramos, este titulo es el que aparece en el tab del navegador y ademas cuando se hace un bookmark

  • <body></body>: Dentro de este elemento se aloja todo el contenido que se va a mostrar en la pagina, ya sea imagenes, texto, videos, videojuegos, etc.

Whitespace in HTML

Para explicar esto pondremos dos ejemplos:

  1. <p id="noWhitespace"> Dogs are silly. </p> 

  2.         
    
        <p id="whitespace"> Dogs
              are
                                  sily </p>

Los dos ejemplos dara el mismo resultado, ya que el parser de HTML reducira los espacios en blanco a uno solo, la razon para dejar mas de un espacio es para que tu codigo sea mas legible.

Entity references: Including special characters in HTML

En HTML hay caracteres que son parte de la sintaxis del lenguaje, como por ejemplo < o &, si quieres usar esto en tu texto y que no se interprete como parte de la sintaxis debes usar una referencia a ese caracter, estas referencias empiezan con un & y terminan con el semicolon (;). Algunos ejemplos son el signo < se hace poniendo la cadena lt entre los signos ya mencionados.
